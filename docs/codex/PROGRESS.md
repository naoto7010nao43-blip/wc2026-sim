- **Methodology finding for Codex's review (not fixed, needs a judgment call): the prediction-benchmark scripts that justified today's rank75 calibration always assign the FIFA-better-ranked team as "home" in every synthetic matchup** (`build_prediction_benchmark_baseline.py`'s `build_matchup_row`, and the same convention reused by `build_aggregation_calibration_sandbox.py`/`build_rating_probe_sensitivity.py`/`build_team_rating_component_audit.py`). The production model applies a constant `config.home_advantage = 0.05` log-lambda bonus to whichever team is passed as "home" (`model_config.py:20` describes it as "a small generic fixture-order edge, not host-nation specific" -- i.e. it's meant to model an arbitrary fixture-order effect, not a rank-correlated one). Verified directly with a one-off in-memory check (not committed) across the same 190 top-20-team matchups used in the real benchmark: `implausible_favorite_count` is 27 when the favorite is always home (matches the committed benchmark), but jumps to 57 when the favorite is always away instead -- 30 individual matchups flip from pass to fail purely based on which side gets the arbitrary home slot. Confirmed this is isolated to the **benchmark/evidence scripts only**: the live tournament simulator (`app/services/tournament.py`'s `play()`) takes `home_team_id`/`away_team_id` from the actual group-draw/bracket fixture order, not from FIFA rank, so the already-smoke-tested live tournament and Monte Carlo odds are unaffected. Because both the `poisson-v1` baseline and the `poisson-v2-rank75` benchmark runs apply this same favorite-as-home convention consistently, the *relative* improvement claim (the 5-case watchlist reduction) is probably still a valid directional signal. But the *absolute* "rank75 benchmark status: pass" headline carries a much wider error bar than it appears -- the true implausible-favorite rate under realistic (rank-independent) home/away assignment likely sits somewhere between 27 and 57, not pinned at 27. Recommend Codex decide the right fix (e.g. average both home/away orderings per matchup, or drop `home_advantage` entirely from the benchmark's lambda calculation) via an explicit spec rather than Claude Code unilaterally changing benchmark methodology.
