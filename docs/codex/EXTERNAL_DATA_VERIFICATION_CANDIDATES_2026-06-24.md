# External Data Verification Candidates — 2026-06-24

Claude Codeが単独作業の一環として、Web検索により実世界の最新データ（監督・フォーメーション・戦術・主力選手の状況・直近フォーム）を調査し、シミュレーションの既存シードデータと照合した結果です。

**このファイルは読み取り専用の候補レポートであり、`backend/data/seed/*.json`・能力値・フォーミュラ・プログラムコードは一切変更していません。** `docs/codex/DATA_GOVERNANCE_POLICY.md`に従い、適用するかどうかの判断はすべてCodexに委ねます。

## 調査スコープと優先順位の根拠

全48ヶ国を同じ深さで調査するのは非現実的なため、既存の診断レポートが最も強い「要確認」信号を出している8チームを選びました：

- `squad_rating_gap_review`の上位4チーム（実質的なランク対比モデル評価の見劣り）: **CRO, NED, POR, MEX**
- `team_rating_component_audit`の`rank_signal_far_above_squad_strength`上位4チーム（FIFAランクに対してスカッド評価が最も圧縮されているチーム）: **ARG, ESP, ENG, FRA**

各チームについて、Web検索を使う独立したサブエージェント（モデル: Opus、複雑な実態判断が必要なため）に、現在のシード値（監督名・デフォルトフォーメーション・戦術プロファイル・上位5選手の能力値）を提示し、実世界の最新情報と照合させました。

## 調査対象外にした項目：選手交代（重要な構造的注記）

選手交代について実データを調査する前に、エンジンの実装を確認したところ、**現在のシミュレーションには監督別の選手交代傾向というパラメータが一切存在しない**ことが分かりました（`backend/app/engine/management.py`の`maybe_substitute`）。全48チームが「疲労した先発選手を、ベストオーバーオールのベンチ選手に機械的に交代する」という共通ロジックで動いており、監督ごとの早め/遅めの交代傾向、同ポジション交代の好み、攻撃的交代の傾向などは一切モデル化されていません。

そのため、実世界の選手交代パターンを調査しても**今は反映先がなく**、データ更新候補ではなく**新機能の提案**になります。今回は時間を選手能力・フォーメーション・戦術・主力選手状況の調査に割り当て、選手交代は「将来のCodex spec候補」として末尾に記載するに留めました。

## 横断的な分析（Claude Codeの考察）

8チーム分の調査結果を並べて見ると、個別の事実訂正以上に、**3つの繰り返されるパターン**が浮かび上がりました。これは単発の修正候補よりも、Codexにとって構造的な優先順位付けの材料になると考えます。

### パターン1: 「主力リストの守備/古参偏重」が複数チームで共通発生
8チーム中6チーム（ESP, ENG, FRA, NED, MEX, そして程度の差でCRO）で、**シードのトップ5選手リストに、実際にはチームの中核とされる選手（多くは中盤の組み立て役か、最新のブレイク株）が欠けている**という同型のギャップが見つかりました。具体例：スペインのRodri/Pedri/Oyarzabal、イングランドのBellingham、フランスのTchouameni/Kante、オランダのde Jong/van Dijk、メキシコのMora/Alvarez。
→ これは個々の選手データの誤りというより、**選手のシード投入時期と現在の「誰が中核か」の評価がズレていく構造的な鮮度劣化**に見えます。「overallが高い選手=トップ5」という選び方自体は正しいはずなので、むしろ**中盤の組み立て役やアンカー役の選手のoverallが、得点関与の少なさゆえに系統的に低めに出ている**可能性も考えられます（Rodriやde Jongのようなアンカー型MFは、ゴール/アシストに直結しないが実際の重要度は非常に高い）。これは`backend/app/rating/formulas.py`の`OVERALL_WEIGHTS`がMID区分で`shooting`に15%・`passing`に30%を割いている設計と関係するかもしれず、もし本当に構造的なら能力値そのものより**OVERALL_WEIGHTSの妥当性を再検証する価値がある**という仮説です（ただしこれは私からは変更せず、仮説の提示のみです）。

### パターン2: 戦術プロファイル（press_intensity / possession_style / defensive_line_height）が複数の強豪チームで実態より低めに出ている
ESP, ENG, POR, CRO, NEDの5チームで、複数の独立した分析記事が「実態はHigh」と評しているのに、シード値は中〜やや高程度に留まっているケースが繰り返し見つかりました。一方で逆方向（実態より高すぎる）の指摘はARGとENGのdefensive_line_height、CRO/MEXのpress_intensityなど散発的でした。
→ サンプルが8チームと少ないため確定的ではありませんが、**「トップ強豪チームの戦術強度が全体的に少し控えめに見積もられている」という方向性の偏りがあるかもしれない**、という仮説として提示します。これも私からは変更しません。

### パターン3: クラブ移籍・負傷情報の鮮度がチームによってばらつく
Cucurella（ESP）、Guehi（ENG）、Malen（NED）、Gimenez（MEX）は明確に**所属クラブが変わっている**のに古いままの疑いがあり、Timber（NED）・Nico Williams（ESP）・Arteaga（MEX）は**出場可否そのものに直接影響する負傷/落選**が反映されていない疑いがあります。これは個別チームの問題ではなく、**シード更新の頻度・タイミングの問題**に見えます。`metadata.json`の`freshnessPolicy`では`clubMinutesMaxAgeHours`系の鮮度基準が定義済みなので、今回見つかった個別の不一致は、その基準に対する実際の運用が追いついていない具体例として使えると思います。

## 新機能の提案（データではなく仕組みの提案）

- **選手交代の監督別モデリング**: 現状は全チーム共通の疲労ベース交代ロジックのみ。実際の監督ごとの交代傾向（早め/遅め、同ポジション温存型、攻撃的カード）を反映する仕組みがあれば、より現実的な後半の試合展開になる可能性があります。今回は実データ調査の対象外としましたが、将来のspec候補として記録します。

## 限界・情報不足の明示

- 各エージェントの調査はWeb検索ベースであり、一次ソース（公式発表）と二次ソース（分析記事・SNS集計）が混在しています。各チームレポート内で「情報源間の矛盾」「情報不足」と明記された箇所は、追加確認が必要です。
- 戦術プロファイル（0-99スケール）に直接対応する公開の定量指標は存在しないため、「High/Medium/Low」という定性評価からの逆算であり、確定的な数値ではありません。
- 残り40チームは未調査です。今回の8チームは既存診断の「最も要確認」信号に基づく優先順位であり、他のチームにも同様の鮮度劣化が広く存在する可能性があります。
