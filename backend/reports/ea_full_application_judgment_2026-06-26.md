# EA FC 26 全選手適用 — 洗い出し・精度判断・既存データとの関係づけ

**Date:** 2026-06-26
**Question (user):** ワールドカップ出場国の選手を全て洗い出し、適応した方が精度向上に
つながるかどうか判断し、既存の能力のデータとどう関係づけて反映させるか判断して。

---

## 0. データ源と検証 (governance)

- 一括ソース: `github.com/ismailoksuz/EAFC26-DataHub` `players.csv`
  (fifa_version=26, update 2025-09-19, 18,405 名, 110 列 — 6基礎+詳細+GK能力)。
- **検証:** 既に EA 公式ページで手動確認済みの 20 名 × 7 値 = **140 値が完全一致**
  (overall/PAC/SHO/PAS/DRI/DEF/PHY)。よってこの CSV は EA 公式値の正確なプロキシ
  として扱える。`dataConfidence="external"`（"official" にはしない）、各エントリに
  `eaPlayerId`（=EA/sofifa ID, 検証キー）と `sourceDataset` を記録。

## 1. 洗い出し（全量）

669 名 / 48 カ国。EA(FC26) 照合可否で 3 層に分かれた:

| 層 | 人数 | 内容 |
|---|---|---|
| **EA 適用（external）** | **473** | 名前+年齢+ポジション照合で安全に一致。418 outfield + 55 GK |
| **照合不能 → 較正（calibrated）** | **196** | FC26 に存在しない/弱い照合。107 名前不一致 + 73 弱照合 + 16 GK誤一致を安全側で除外 |
| 合計 | 669 | |

照合は名前のアクセント正規化（Côte d'Ivoire / Türkiye）+ 年齢±1 + GK整合で誤一致を排除。
誤一致の典型例 `KOR_KIMSEUNGGYU`（GK が同姓の野手にヒット）は正しく除外。

## 2. 精度向上の判断 — 「適用した方が良いか」

**結論: 照合できる 473 名は適用が大幅な精度向上。ただし全 669 への一律適用は不可能で、
ハイブリッドが必須。**

根拠 — 自前推定 vs EA の重なり（473 ペア）:

- 推定は EA に対し **系統的に中央値 +20 低い**（全品質帯で。トップだけの問題ではない）。
- かつ相関 **0.64** と**ノイズも大きい**（推定 50–54 の選手の EA 実値は 62–91 に散らばる）。
- → EA 適用は「圧縮の解消」だけでなく、較正では回復不能な**個人単位の誤差も是正**する。
  これは推定の再スケールでは得られない、実データだけの利得。

一方:

- **照合不能 196 名は EA に存在しない**（Neymar すら不在。Iran は FC26 全体で 6 名のみ、
  Jordan 3、Uzbekistan 5、South Africa 16、Egypt 10 ＝ 国内リーグ主体の代表は EA が薄い）。
  これらに EA を当てることは原理的に不可能 → 推定を残すしかない。
- 放置すると、薄い国（Iran 等）は「EA の数名 + 推定の十数名（20pt 低い）」で**チーム全体が
  過小評価**され、シミュレーションが不公平になる。

## 3. 既存データとの関係づけ（方法論）— 2 層構成

両者を**一つの EA 基準スケール**に載せて連続させる。

### Layer 1 — EA 注入（473名, `dataConfidence=external`）
EA の overall + 6基礎値（GK は REF/HAN/SPD）を**verbatim** で base に採用し、詳細サブ能力は
従来式で派生（内部整合・エンジン側の特別扱い不要）。uncertainty=0.05。

### Layer 2 — EA 基準への較正（196名, `dataConfidence=estimated` + `calibrationApplied`）
473 の重なりから **単調写像 est_overall → EA_overall** を等調回帰(PAV)で推定し、各推定選手の
base を EA スケールへシフト（overall とサブ能力が一緒に動くので整合を保つ）。

**正直さのための 2 つの抑制:**
- **SHRINK=0.7** … 照合不能選手は弱いリーグ寄りの偏り（選択バイアス）があり、同じ推定値の
  照合選手より実際は少し弱い可能性が高い。よって当てはめ差分を全部ではなく一部だけ適用。
- **MAX_SHIFT=20** … 極端な外挿を抑制。
- これらは推定（実データではない推論）なので uncertainty を +0.05 し、`calibrationApplied=True`
  を API/サイトに surface（プレーン推定と区別表示可能）。

## 4. 結果（適用後の分布）

| | before | after |
|---|---|---|
| overall 範囲 | 38–91（85 以上は EA の 11 名だけ、70–84 が空洞） | **58–91, 中央値 75** |
| 70–84 バンド | 約 13 名 | **392 名**（空洞解消、連続） |
| Iran 平均 | 推定のみで ~50 台に潰れる | **65.6**（Taremi 78=EA, 他 60–68 較正）妥当な中堅代表 |

marquee は EA 完全一致（Haaland 90 / Mbappé 91 / Pedri 89 / Raphinha 89 / Upamecano 85）。

## 6. 残課題（次段階・任意）
- 73 名の「弱照合」は安全側で較正に回している。手動で EA ページを確認すれば一部は external に
  昇格でき、その分だけ精度がさらに上がる（市場価値上位から順に消化）。
- SHRINK/MAX_SHIFT は当面 0.7 / 20。実シミュ結果（薄い国の勝率）を見て微調整余地。
